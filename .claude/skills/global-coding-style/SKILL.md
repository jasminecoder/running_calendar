## When to use this skill

- When writing any Ruby code (`.rb` files)
- When naming variables (snake_case), methods (snake_case), and classes (CamelCase)
- When structuring methods with guard clauses (early returns)
- When using Ruby 3.1+ hash shorthand `{ name:, date: }`
- When deciding on string literal style (double quotes)
- When keeping methods short and focused (under 10 lines)
- When removing dead code, commented blocks, or unused imports
- When refactoring for clarity and readability
